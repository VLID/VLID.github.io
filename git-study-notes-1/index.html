前言

起因

知道Git这个名字应该是很早之前的事了，大概是三四年前，但真正明白和开始接触Git却是在一年前的Group Project中。当时小组里有个美国大神，在他的指导下我们几个成员也开始使用Git。那时只是会4个基本的commands：$ git add $ git commit -m $ git push $ git pull。

近期在做iOS的开发，为了更好的控制代码版本和修改代码的方便性，决定将代码托管到Github.com，就是这样一个契机让我决定要扎实学好Git。

重拾Git

一直以来都想深入扎实的自学下Git命令，却一直处在入门阶段，因为没有好好地总结和回顾，再加上平时确实没有将Git用于实际，总处于临时用临时看，过后不用就忘，忘了后再临时搜索。如此反复，很痛苦更浪费时间。于是痛定思痛，决心与其浪费时间做重复工不如花费些时间好好总结，将来复习自己的东西能快速唤起记忆。

声明

本人对Git的学习是基于Mac平台，使用的是原生命令行模式，即Command Line。如对Command Line不熟悉，可自行Google快速入门。

何为Git？

Git是一个分布式版本控制系统，与分布式相对的是集中式，即SVN。而Git的优势之一在于分布，不需要服务器就可以运作版本控制，于是每个人的电脑就都可以是一个完整的版本库。因此Git更像是文件系统，直接在本地就可以获取想要的版本，而不必连接到服务器端获取。

那什么是版本控制？假设你开发一个软件，第一天写了三段代码，第二天发现第二段代码可能没必要写于是删除了，但到了第三天又发现其实第二段代码还是会在之后的开发中使用到的于是又想恢复回来，但这时发现原先的第二段代码已删的一干二净。你可能想：我可以把每天修改的文件单独另存啊。但你有没有想过，一旦日子长了、文件复杂庞大了，当你想要恢复到之前的某一个版本时，是不是就变得麻烦复杂多了？所以我们需要结束这种手动管理多个不同的版本的做法，让这一切交给Git来做。

Git是由Linux之父 - Linus Torvalds创作，并于2005年发布。Git的最初目的是为了更好地管理Linux内核开发而设计的（Git开发的精彩小故事，延伸阅读：Git的诞生 by 廖雪峰）。

安装Git

从Git官网下载相应平台的安装包直接安装即可。

Git配置(config)指令

安装后首先需要完成两个配置指令，因为每个机器都需要自报家门。（注意：用了--global参数，意味着这台机器上所有的Git仓都会使用这个配置，如果你想针对特定项目使用不同的用户名和邮箱，可以在那个项目目录下运行没有--global参数的下列指令即可）

  1. $ git config --global user.name "[name]"
    对commit操作设置关联的用户名

  2. $ git config --global user.email "[email]"
    对commit操作设置关联的邮箱地址

如果想查看所有的配置信息，可输入：
$ git config --list

如果想单独查看用户名，可输入：
$ git config user.name

如果想单独查看邮箱，可输入：
$ git config user.email

以此类推：可以通过输入 $ git config <key> 来检查Git的某一项配置。

Git帮助（初学者的好帮手）

获取帮助的过程就是学习的过程，所以懂得如何在Git中获取帮助显得尤为重要。有三种方法可以获得Git帮助：

1. `$ git help <verb>`
2. `$ git <verb> --help`
3. `$ man git-<verb>`

例如，你想获得更多的config的用法，就可以执行：
$ git help config

如果你想了解有哪些<verb>，可以直接输入：
$ git help

Git创建仓库(repository)指令

首先需要做的是找到一个已经存在的本地文件夹或者通过Terminal执行以下Command Line创建新的文件夹：

$ mkdir "文件夹名"
$ cd "文件夹名"

在Terminal中进入新建文件夹后，接着输入：
$ git init
此命令可将新建文件夹变成Git可以管理的仓库，即创建仓库。同时在这个文件夹中也会生成名为.git的子文件夹，这个文件夹包含有初始化的Git仓库中所有的必须文件。

如果想要获得一份已经存在的Git仓库，你可以使用clone指令：

  1. 当你想创建一个本地仓库的克隆版本：

    $ git clone /path/to/repository

  2. 当你想获得一个远端仓库的克隆版本：

    ·方法一
    $ git clone username@host:/path/to/repository

    以Github.com上的Swift项目为例：
    $ git clone git@github.com:apple/swift.git

    ·方法二
    $ git clone [url]

    以Github.com上的Swift项目为例：
    $ git clone https://github.com/apple/swift.git

    如果想自定义本地仓库名，可以输入：
    $ git clone https://github.com/apple/swift.git myswift
